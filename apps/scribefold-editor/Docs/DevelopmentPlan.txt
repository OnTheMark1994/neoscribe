Overal plan and timeline for the development of the application.

Then and Now

    Previously
        did not create a plan
        did not lay out file structure, data flows, user stories etc before
        did not set timelines

        so always pushing as fast as I can
        cutting cornders and moving on to next thing as soon as current on at minimum viable 
        unsustainably spending time on it until burning out

        put everything else on pause to fosuc on it entirely 
        and it was so long all at once it could not be completed before other thigns piled up and needed to stop
        if didn't complete the entire thing it was a failure

        trying to do the whole thing at once in one big bite
        can't eat an entire steak in one bite
        but you can break it into many piecs and finish it fine

    Better:
        create a plan for phases
        create a timeline for each phase
        do not progress past that so it stays sustainable
            when you are done with the weeks work you do other things that need to be done and to balance life so you can carry on
        if you spend the number of weeks on it you predict you will have the finished product at the time you expect



        Each phase has a part that is complete
        typing out the user interactions stream of consciousness
        while doing that typing out the components with their displays and state interactions
        and in the 3rd window have the file structure 
        create files based on files strucutre
        put psudocode notes at the top of each
        fill them in (ai can do this becaues its by your plan and you can do in small increments looking it over for spaghetti)
